# 异常处理规范

> 本文档定义 OpenClawHub 的统一异常处理规范，参考 SpringBoot 风格。

## 1. 响应格式

所有 API 响应遵循统一格式：

```json
{
  "code": 0,
  "message": "success",
  "data": {...}
}
```

### 成功响应
- `code`: 0
- `message`: "success"
- `data`: 业务数据

### 错误响应
- `code`: 非 0，错误码
- `message`: 错误描述
- `data`: null 或详细错误信息

## 2. HTTP 状态码

| HTTP Status | 含义 | 使用场景 |
|-------------|------|---------|
| 400 | Bad Request | 请求参数错误 |
| 401 | Unauthorized | 未认证或认证失败 |
| 403 | Forbidden | 无权限访问 |
| 404 | Not Found | 资源不存在 |
| 409 | Conflict | 资源冲突 |
| 422 | Unprocessable Entity | 验证错误 |
| 500 | Internal Server Error | 服务器内部错误 |

## 3. 错误码体系

错误码格式：`{HTTP_STATUS}{SEQUENCE}`

例如：`40401` = 404 (Not Found) + 01 (序号)

| 错误码 | HTTP Status | 含义 | 说明 |
|--------|-------------|------|------|

| 40303 | 403 | Cannot Remove Owner | 不能移除项目所有者 |
| 40304 | 403 | Only Owner Can Revoke | 只有所有者可以撤销邀请 |
| 40401 | 404 | Not Found | 资源不存在 |

| 40403 | 404 | Agent Not Found | Agent 不存在 |
| 40404 | 404 | User Not Found | 用户不存在 |

### 5xx Server Errors

| 错误码 | HTTP Status | 含义 | 说明 |
|--------|-------------|------|------|
| 50001 | 500 | Internal Error | 一般内部错误 |
| 50002 | 500 | Database Error | 数据库错误 |
| 50003 | 500 | External Service Error | 外部服务错误 |

## 5. 使用示例

### 在 Service 中抛出异常

```python
from core.exceptions import ProjectNotFoundException, ForbiddenException

class ProjectService:
    def get_project(self, project_id: str):
        project = self.db.query(Project).filter(Project.id == project_id).first()
        if not project:
            raise ProjectNotFoundException(project_id)

    def update_project(self, project, user):
        if not self.is_admin(project, user):
            raise ForbiddenException(message="Admin role required")
```

### 全局异常捕获

`main.py` 中配置了全局异常处理器：

```python
@app.exception_handler(Exception)
async def general_exception_handler(request: Request, exc: Exception):
    """未捕获的异常返回标准化 500 错误"""
    logger.error(f"Unhandled exception: {exc}")
    return JSONResponse(
        status_code=500,
        content={
            "code": ErrorCodes.INTERNAL_ERROR,
            "message": "Internal server error",
            "data": None
        }
    )
```

## 6. 错误响应示例

### 404 Not Found
```json
{
  "code": 40402,
  "message": "Project not found: abc-123",
  "details": null
}
```

### 403 Forbidden
```json
{
  "code": 40302,
  "message": "Admin or owner role required. Required roles: owner, admin",
  "details": null
}
```

### 422 Validation Error
```json
{
  "code": 42201,
  "message": "Validation error",
  "details": [
    {"loc": ["body", "email"], "msg": "Invalid email format"}
  ]
}
```

### 500 Internal Error
```json
{
  "code": 50001,
  "message": "Internal server error",
  "data": null
}
```

## 7. 路由层异常处理

路由层仍使用 `HTTPException`，但推荐使用统一的 `AppException` 子类：

```python
from fastapi import APIRouter, Depends, HTTPException
from core.exceptions import ProjectNotFoundException

router = APIRouter()

@router.get("/{project_id}")
def get_project(project_id: str, service: ProjectService = Depends()):
    try:
        return service.get_project(project_id)
    except ProjectNotFoundException:
        raise  # AppException 会被全局处理器捕获
```

## 8. 日志记录

未捕获的异常会被记录到日志：

```python
logger.error(f"Unhandled exception: {exc.__class__.__name__}: {str(exc)}")
logger.error(traceback.format_exc())
```
