# OpenClawHub Skills 索引

> 角色 Skills 目录，用于 AI 助理在执行不同角色任务时加载对应规范

## 角色列表

| 角色 | 目录 | 职责 |
|------|------|------|
| **产品经理** | `product-manager/` | 产品规划、PRD、需求分析 |
| **项目经理** | `pm/` | 项目规划、进度管理、跨角色协调 |
| **UI设计师** | `ui-designer/` | 界面设计、交互设计、视觉规范 |
| **开发工程师** | `developer/` | 架构设计、核心模块开发 |
| **前端开发** | `frontend/` | Vue3 页面和组件开发 |
| **后端开发** | `backend/` | FastAPI 服务和数据库开发 |
| **测试工程师** | `tester/` | 测试计划、用例、测试执行 |

## 协作关系

## 知识库查询规则总结

| 角色 | 查询内容 |
|------|----------|
| **产品经理** | 知识库索引 + OpenClawHub 相关文档 + PRD |
| **项目经理** | 知识库索引 + OpenClawHarness 实现文档 + 复盘文档 |
| **UI设计师** | 产品经理审批文档 + 项目经理审批文档 |
| **开发工程师** | 知识库技术文档 + corrections 历史经验 |
| **前端开发** | 开发文档 + API 规范 + 审批通过文档 |
| **后端开发** | 开发文档 + API 规范 + 现有代码结构 |
| **测试工程师** | 产品需求文档 + 开发文档 + API 路由 + 页面 |

## 审批流程

1. **文档提交** → 各角色完成文档后提交给对应审批者
2. **PM/产品经理审核** → 审核需求和设计文档
3. **项目经理审核** → 审核项目计划和进度
4. **CEO（老赵）终审** → 最终审批
