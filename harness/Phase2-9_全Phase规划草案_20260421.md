## Phase 3: Agent 管理模块（高级）

### 定位
Phase 2 实现了 Agent 的 CRUD，Phase 3 扩展到 Agent 的**高级配置、角色定义、技能绑定**。

### 功能清单

#### 后端 API（~20 个）

**Agent 角色与技能 (8 个)**
- `GET /api/agent-roles` - Agent 角色列表
- `POST /api/agent-roles` - 创建角色模板
- `GET /api/agent-roles/{id}` - 角色详情
- `PUT /api/agent-roles/{id}` - 更新角色
- `DELETE /api/agent-roles/{id}` - 删除角色
- `POST /api/agents/{id}/skills` - 给 Agent 绑定技能
- `DELETE /api/agents/{id}/skills/{skill_id}` - 解绑技能
- `GET /api/agents/{id}/skills` - Agent 的技能列表

**Agent 记忆与上下文 (6 个)**
- `GET /api/agents/{id}/memory` - Agent 记忆配置
- `PUT /api/agents/{id}/memory` - 更新记忆配置
- `POST /api/agents/{id}/context` - 设置 Agent 上下文
- `GET /api/agents/{id}/history` - 对话历史
- `DELETE /api/agents/{id}/memory/clear` - 清除记忆
- `POST /api/agents/{id}/reset` - 重置 Agent 状态

**Agent 监控指标 (6 个)**
- `GET /api/agents/{id}/metrics` - Agent 运行指标
- `GET /api/agents/{id}/metrics/daily` - 每日统计
- `GET /api/agents/{id}/tasks/count` - 已完成/进行中任务数
- `GET /api/orgs/{id}/agents/usage` - 组织内 Agent 使用统计
- `GET /api/agents/{id}/performance` - Agent 性能报告
- `GET /api/agents/{id}/health` - Agent 健康检查

#### 数据库模型

**agent_roles 表**
```python
- id: UUID (PK)
- name: VARCHAR(64)
- description: TEXT
- system_prompt_template: TEXT
- default_config: JSON  # 默认配置
- org_id: UUID (FK)
- created_at, updated_at
```

**agent_skills 表**
```python
- id: UUID (PK)
- agent_id: UUID (FK)
- skill_name: VARCHAR(128)
- skill_config: JSON
- enabled: BOOLEAN
- created_at
```

**agent_metrics 表**
```python
- id: UUID (PK)
- agent_id: UUID (FK)
- date: DATE
- tasks_completed: INT
- tasks_failed: INT
- avg_response_time_ms: INT
- token_usage: INT
- created_at
```

#### 前端页面（5 个）

- `/agents/:id/config` - Agent 高级配置页
- `/agents/:id/memory` - Agent 记忆管理页
- `/agents/:id/metrics` - Agent 指标仪表板
- `/agent-roles` - 角色模板管理页
- `/agent-roles/new` - 创建角色模板页

### 验收标准
- [ ] Agent 角色模板系统完整
- [ ] Agent 技能绑定/解绑正常
- [ ] Agent 记忆配置可更新
- [ ] 指标统计正常

### 风险点
- Agent 记忆系统的实现复杂度（需要了解 Hermes Agent 的记忆接口）
- 指标采集频率和存储成本

## Phase 4: 任务看板模块

### 定位
Phase 4 是核心**协作功能**，让 Agent 和人能在同一个任务看板上工作。

### 功能清单

#### 后端 API（~25 个）

**任务 CRUD (8 个)**
- `POST /api/tasks` - 创建任务
- `GET /api/tasks` - 任务列表（支持 filter: status, assignee, project）
- `GET /api/tasks/{id}` - 任务详情
- `PUT /api/tasks/{id}` - 更新任务
- `DELETE /api/tasks/{id}` - 删除任务
- `POST /api/tasks/bulk` - 批量创建任务
- `PUT /api/tasks/bulk/status` - 批量更新状态
- `GET /api/tasks/export` - 导出任务（CSV/JSON）

**任务协作 (8 个)**
- `POST /api/tasks/{id}/assign` - 分配任务（给人或 Agent）
- `POST /api/tasks/{id}/claim` - 认领任务
- `POST /api/tasks/{id}/complete` - 完成任务
- `POST /api/tasks/{id}/comment` - 添加评论
- `GET /api/tasks/{id}/comments` - 评论列表
- `POST /api/tasks/{id}/subtasks` - 创建子任务
- `GET /api/tasks/{id}/subtasks` - 子任务列表
- `POST /api/tasks/{id}/attachments` - 上传附件

**看板视图 (6 个)**
- `GET /api/projects/{id}/kanban` - 项目看板数据
- `GET /api/projects/{id}/tasks/by-status` - 按状态分组
- `GET /api/projects/{id}/tasks/by-assignee` - 按负责人分组
- `GET /api/projects/{id}/tasks/timeline` - 时间线视图
- `POST /api/tasks/{id}/move` - 移动任务（看板拖拽）
- `GET /api/tasks/{id}/activity` - 任务活动历史

**提醒与到期 (3 个)**
- `POST /api/tasks/{id}/remind` - 设置提醒
- `GET /api/tasks/due-soon` - 即将到期任务
- `POST /api/tasks/{id}/snooze` - 稍后提醒

#### 数据库模型

**tasks 表（补充字段）**
```python
- parent_id: UUID (FK, nullable)  # 父任务（子任务）
- root_id: UUID (FK, nullable)    # 根任务（跨层级）
- position: INT                  # 看板顺序
- estimated_hours: FLOAT
- actual_hours: FLOAT
- priority: ENUM('low', 'medium', 'high', 'urgent')
- tags: JSON  # 标签数组
- reminder_at: DATETIME (nullable)
```

**task_comments 表**
```python
- id: UUID (PK)
- task_id: UUID (FK)
- user_id: UUID (FK)
- content: TEXT
- created_at, updated_at
```

**task_attachments 表**
```python
- id: UUID (PK)
- task_id: UUID (FK)
- filename: VARCHAR(256)
- file_url: VARCHAR(512)
- file_size: INT
- uploaded_by: UUID (FK)
- uploaded_at
```

#### 前端页面（6 个）

- `/projects/:id/kanban` - 看板视图（核心）
- `/projects/:id/tasks` - 任务列表视图
- `/projects/:id/timeline` - 时间线视图
- `/tasks/:id` - 任务详情抽屉/弹窗
- `/tasks/:id/edit` - 任务编辑页
- `/tasks/due-soon` - 即将到期任务页

#### 前端组件

- KanbanBoard.vue - 看板主组件
- KanbanColumn.vue - 看板列
- TaskCard.vue - 任务卡片（可拖拽）
- TaskDetailDrawer.vue - 任务详情抽屉
- CommentThread.vue - 评论组件
- SubtaskList.vue - 子任务列表
- DueDatePicker.vue - 到期日选择器
- TaskFilter.vue - 任务筛选器

### 验收标准
- [ ] 看板拖拽排序正常
- [ ] 任务分配给人和 Agent 正常
- [ ] 子任务支持嵌套
- [ ] 评论功能正常
- [ ] 多视图切换正常（看板/列表/时间线）

### 风险点
- 前端拖拽库选型（建议用 @vueuse/integrations + sortablejs）
- 任务量大时的性能问题（需要分页/虚拟滚动）
- WebSocket 实时同步看板状态

## Phase 5: 执行引擎集成

### 定位
Phase 5 是**自动化核心**，让任务真正被 Agent 执行。需要与 Hermes Runtime 深度集成。

### 功能清单

#### 后端 API（~15 个）

**执行引擎 (8 个)**
- `POST /api/tasks/{id}/execute` - 触发任务执行
- `POST /api/tasks/{id}/execute/batch` - 批量执行
- `GET /api/executions/{id}` - 执行记录详情
- `GET /api/tasks/{id}/executions` - 任务执行历史
- `POST /api/executions/{id}/cancel` - 取消执行
- `POST /api/executions/{id}/retry` - 重试执行
- `GET /api/executions/{id}/output` - 执行输出
- `GET /api/executions/active` - 当前活跃执行

**Agent 调度 (4 个)**
- `POST /api/scheduler/jobs` - 创建定时任务
- `GET /api/scheduler/jobs` - 定时任务列表
- `DELETE /api/scheduler/jobs/{id}` - 删除定时任务
- `GET /api/scheduler/jobs/{id}/runs` - 执行记录

**工作流 (3 个)**
- `POST /api/workflows` - 创建工作流
- `GET /api/workflows/{id}` - 工作流详情
- `POST /api/workflows/{id}/execute` - 执行工作流

#### 数据库模型

**executions 表**
```python
- id: UUID (PK)
- task_id: UUID (FK)
- agent_id: UUID (FK)
- status: ENUM('pending', 'running', 'completed', 'failed', 'cancelled')
- input_data: JSON
- output_data: JSON
- error_message: TEXT
- started_at: DATETIME
- completed_at: DATETIME
- created_at
```

**scheduler_jobs 表**
```python
- id: UUID (PK)
- name: VARCHAR(128)
- task_template_id: UUID (FK)
- cron_expression: VARCHAR(64)
- agent_id: UUID (FK)
- enabled: BOOLEAN
- last_run_at: DATETIME
- next_run_at: DATETIME
- created_at
```

**workflows 表**
```python
- id: UUID (PK)
- name: VARCHAR(128)
- description: TEXT
- steps: JSON  # 工作流步骤定义
- org_id: UUID (FK)
- created_by: UUID (FK)
- created_at, updated_at
```

#### 前端页面（4 个）

- `/executions` - 执行记录页
- `/executions/:id` - 执行详情页
- `/scheduler` - 定时任务管理页
- `/workflows` - 工作流编辑器页

#### 前端组件

- ExecutionLogViewer.vue - 执行日志查看器
- WorkflowEditor.vue - 工作流可视化编辑器
- SchedulerConfig.vue - 定时配置组件
- ExecutionStatus.vue - 执行状态指示器

### 验收标准
- [ ] 任务能触发 Agent 执行
- [ ] 执行状态实时更新（WebSocket）
- [ ] 执行日志完整记录
- [ ] 定时任务正常调度
- [ ] 工作流能定义和执行

### 风险点
- **Hermes Runtime API** - 需要确认 Hermes 的执行管理接口
- 执行超时处理
- 并发执行控制
- 工作流的状态机设计

## Phase 6: Activity Feed

### 定位

### 功能清单

#### 后端 API（~8 个）
- `GET /api/activities` - 全局活动流

- `POST /api/activities/mark-read` - 标记已读

#### 数据库模型

**activities 表**
```python
- id: UUID (PK)
- org_id: UUID (FK)
- project_id: UUID (FK, nullable)
- actor_type: ENUM('user', 'agent', 'system')
- actor_id: UUID
- action: VARCHAR(64)  # 'task.created', 'agent.started', etc.
- target_type: VARCHAR(32)
- target_id: UUID
- metadata: JSON
- is_read: BOOLEAN
- created_at
```

#### 前端页面（2 个）
- `/activities` - 活动通知页

#### 前端组件
- ActivityFeed.vue - 活动流组件
