## 一、Phase 2 定位

**Phase 2 = 项目管理 + Agent 管理**

Phase 1 搭建了「用户-组织」基础，Phase 2 扩展到「项目-Agent」核心业务。

**业务逻辑链：**
```
用户 (Phase 1) → 组织 (Phase 1) → 项目 (Phase 2) → Agent (Phase 2) → 任务 (Phase 4)
```

## 二、Phase 2 功能清单

### 2.1 后端 API（预计 ~30 个）

#### 项目管理 (10 个)
- `POST /api/projects` - 创建项目
- `GET /api/projects` - 项目列表
- `GET /api/projects/{id}` - 项目详情
- `PUT /api/projects/{id}` - 更新项目
- `DELETE /api/projects/{id}` - 删除项目
- `GET /api/projects/{id}/members` - 项目成员列表
- `POST /api/projects/{id}/members` - 添加项目成员
- `DELETE /api/projects/{id}/members/{user_id}` - 移除项目成员
- `GET /api/projects/{id}/agents` - 项目下的 Agent 列表
- `POST /api/projects/{id}/agents` - 给项目分配 Agent

#### Agent 管理 (8 个)
- `GET /api/agents` - Agent 列表（全局）
- `GET /api/agents/{id}` - Agent 详情
- `PUT /api/agents/{id}` - 更新 Agent 配置
- `DELETE /api/agents/{id}` - 删除 Agent
- `POST /api/agents/{id}/start` - 启动 Agent
- `POST /api/agents/{id}/stop` - 停止 Agent
- `GET /api/agents/{id}/status` - Agent 状态
- `POST /api/agents/{id}/projects/{project_id}` - Agent 加入项目

#### 任务管理 (8 个) - 可选 Phase 2 或 Phase 4
- `POST /api/tasks` - 创建任务
- `GET /api/tasks` - 任务列表
- `GET /api/tasks/{id}` - 任务详情
- `PUT /api/tasks/{id}` - 更新任务
- `DELETE /api/tasks/{id}` - 删除任务
- `POST /api/tasks/{id}/assign` - 分配任务给 Agent
- `POST /api/tasks/{id}/complete` - 完成任务
- `GET /api/projects/{id}/tasks` - 项目下的任务列表

#### 运行时日志 (4 个)
- `GET /api/agents/{id}/logs` - Agent 运行日志
- `GET /api/tasks/{id}/logs` - 任务执行日志
- `GET /api/projects/{id}/activity` - 项目活动流
- `WS /ws/agents/{id}` - Agent 实时状态 WebSocket

### 2.2 数据库模型

#### projects 表
```python
- id: UUID (PK)
- name: VARCHAR(128)
- description: TEXT (nullable)
- org_id: UUID (FK -> organizations.id)
- status: ENUM('active', 'archived', 'deleted')
- settings: JSON  # 项目配置
- created_by: UUID (FK -> users.id)
- created_at, updated_at
```

#### project_members 表
```python
- id: UUID (PK)
- project_id: UUID (FK)
- user_id: UUID (FK)
- role: ENUM('owner', 'admin', 'member')
- joined_at
```

#### agents 表
```python
- id: UUID (PK)
- name: VARCHAR(64)
- description: TEXT
- agent_type: VARCHAR(32)  # 'hermes', 'claude', 'openai'
- config: JSON  # Agent 配置（模型、温度、系统提示等）
- org_id: UUID (FK -> organizations.id)
- status: ENUM('offline', 'online', 'busy', 'error')
- last_seen_at: DATETIME
- created_at, updated_at
```

#### project_agents 表（多对多）
```python
- id: UUID (PK)
- project_id: UUID (FK)
- agent_id: UUID (FK)
- assigned_at: DATETIME
```

#### tasks 表
```python
- id: UUID (PK)
- title: VARCHAR(256)
- description: TEXT
- project_id: UUID (FK)
- assignee_agent_id: UUID (FK, nullable)
- status: ENUM('pending', 'in_progress', 'completed', 'failed')
- priority: ENUM('low', 'medium', 'high')
- due_date: DATETIME (nullable)
- created_by: UUID (FK)
- created_at, updated_at, completed_at
```

## 三、技术要点

### 3.1 复用 Phase 1 基础设施
- 认证：JWT（继续使用 Phase 1 的认证系统）
- 权限：项目级权限检查（org_id + project_id 双重校验）
- 响应格式：`{ code: 0, message: "success", data: {} }`

### 3.2 Agent 运行时集成
- Agent 创建后，实际启动需要在 Hermes Runtime 中创建实例
- `/api/agents/{id}/start` → 调用 Hermes Runtime API
- Agent 状态通过 WebSocket 实时推送

### 3.3 项目-组织关系
- 项目归属于组织（org_id）
- 项目成员是组织成员的子集
- Agent 归属于组织，项目使用组织的 Agent
