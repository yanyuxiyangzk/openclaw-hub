## PM 第一轮审核问题整改

| # | 问题 | 整改方案 |
|---|------|----------|

## 整体架构演进（整改后）

```
Phase 1          Phase 2          Phase 3          Phase 4          Phase 5
用户/组织   →    项目/Agent   →   Agent配置      →   任务看板      →   执行引擎
认证模块         核心业务          高级配置          协作管理          自动化
                                              ↑
                                              │
                                        人审核确认
```

**核心业务流程：**
```
人(CEO) → 创建任务 → 分配给 Agent → Agent 执行 → 人审核 → 任务完成
```

## Phase 2: 项目管理 + Agent 管理（整改版）

### 定位
Phase 2 是**核心业务起点**，建立 项目-Agent 关系，Agent 仅支持 subprocess 模式运行（无 HTTP API）。

### 功能清单（整改后）

#### 后端 API（~28 个）

**项目管理 (8 个)**
- `POST /api/projects` - 创建项目
- `GET /api/projects` - 项目列表
- `GET /api/projects/{id}` - 项目详情
- `PUT /api/projects/{id}` - 更新项目
- `DELETE /api/projects/{id}` - 删除项目
- `GET /api/projects/{id}/members` - 项目成员列表
- `POST /api/projects/{id}/members` - 添加项目成员
- `DELETE /api/projects/{id}/members/{user_id}` - 移除项目成员

**Agent 基础管理 (10 个)**
- `GET /api/agents` - Agent 列表（全局）
- `GET /api/agents/{id}` - Agent 详情
- `POST /api/agents` - 创建 Agent
- `PUT /api/agents/{id}` - 更新 Agent 配置
- `DELETE /api/agents/{id}` - 删除 Agent
- `GET /api/agents/{id}/status` - Agent 状态（通过 HermesInstanceManager）
- `POST /api/agents/{id}/projects/{project_id}` - Agent 加入项目
- `GET /api/projects/{id}/agents` - 项目下的 Agent 列表
- `GET /api/projects/{id}/agents/available` - 项目可用 Agent（未加入的）
- `DELETE /api/projects/{id}/agents/{agent_id}` - 从项目移除 Agent

**运行时管理 (6 个)**
- `POST /api/agents/{id}/start` - 启动 Agent（subprocess）
- `POST /api/agents/{id}/stop` - 停止 Agent
- `GET /api/agents/{id}/logs` - Agent 运行日志（从文件系统读取）
- `GET /api/agents/{id}/health` - Agent 健康检查
- `GET /api/agents/active` - 当前活跃的 Agent
- `WS /ws/agents/{id}/status` - Agent 状态 WebSocket（自建）

**项目-Agent 关联 (4 个)**
- `POST /api/projects/{id}/agents/assign` - 分配 Agent 到项目
- `GET /api/projects/{id}/tasks` - 项目下的任务列表（Phase 4 预留）
- `POST /api/projects/{id}/tasks` - 创建项目任务（Phase 4 预留）
- `GET /api/projects/{id}/activity` - 项目活动流（Phase 6 预留）

#### 数据库模型（整改后：4 张表）

**projects 表**
```python
- id: UUID (PK)
- name: VARCHAR(128)
- description: TEXT (nullable)
- org_id: UUID (FK -> organizations.id)
- status: ENUM('active', 'archived', 'deleted')
- settings: JSON
- created_by: UUID (FK -> users.id)
- created_at, updated_at
```

**agents 表**
```python
- id: UUID (PK)
- name: VARCHAR(64)
- description: TEXT
- agent_type: VARCHAR(32)  # 'hermes'
- config: JSON  # Agent 配置（系统提示、温度等）
- org_id: UUID (FK -> organizations.id)
- status: ENUM('offline', 'online', 'busy', 'error')
- last_seen_at: DATETIME
- created_at, updated_at
```

**project_agents 表**
```python
- id: UUID (PK)
- project_id: UUID (FK)
- agent_id: UUID (FK)
- assigned_at: DATETIME
- UNIQUE(project_id, agent_id)
```

**project_members 表（复用 Phase 1 的 organization_members 结构）**
```python
- id: UUID (PK)
- project_id: UUID (FK)
- user_id: UUID (FK)
- role: ENUM('owner', 'admin', 'member')
- joined_at
```

#### 前端页面（6 个）

- `/projects` - 项目列表页
- `/projects/new` - 创建项目页
- `/projects/:id` - 项目详情页（包含 Agent 分配）
- `/projects/:id/settings` - 项目设置页
- `/agents` - Agent 列表页
- `/agents/:id` - Agent 详情/配置页

#### 明确不包含（整改删除）

- ❌ HTTP API 包装层（延后到 Phase 5）
- ❌ Agent 角色模板（Phase 3 轻量化处理）
- ❌ WebSocket 实时推送（Phase 5 自建）

#### 验收标准
- [ ] 项目 CRUD 正常

## Phase 4: 任务看板模块（整改版）

### 定位
Phase 4 是**核心协作功能**，实现 人-Agent 任务协作流程。

### 核心业务流程（整改明确）
```
创建任务 → 分配给 Agent → Agent 执行（Phase 5）→ 人审核 → 任务完成
```

**任务状态机：**
```
pending → in_progress → completed → approved
                 ↓            ↓
              failed       rejected
```

| 状态 | 说明 | 操作者 |
|------|------|--------|
| pending | 待分配 | 人 |
| in_progress | 执行中 | Agent |
| completed | 执行完成 | Agent |
| approved | 审核通过 | 人 |
| failed | 执行失败 | Agent |
| rejected | 审核拒绝 | 人 |

### 功能清单（整改后：~22 个 API）

**任务基础 CRUD (8 个)**
- `POST /api/tasks` - 创建任务
- `GET /api/tasks` - 任务列表（filter: status, project, assignee）
- `GET /api/tasks/{id}` - 任务详情
- `PUT /api/tasks/{id}` - 更新任务
- `DELETE /api/tasks/{id}` - 删除任务
- `POST /api/tasks/bulk` - 批量创建
- `PUT /api/tasks/bulk/status` - 批量更新状态
- `GET /api/tasks/export` - 导出任务

**任务协作流程 (8 个)**
- `POST /api/tasks/{id}/assign` - 分配任务（给人或 Agent）
- `POST /api/tasks/{id}/claim` - 认领任务
- `POST /api/tasks/{id}/start` - 开始执行（设置 in_progress）
- `POST /api/tasks/{id}/complete` - 完成执行（设置 completed）
- `POST /api/tasks/{id}/approve` - 审核通过（设置 approved）
- `POST /api/tasks/{id}/reject` - 审核拒绝（设置 rejected）
- `POST /api/tasks/{id}/comment` - 添加评论
- `GET /api/tasks/{id}/comments` - 评论列表

**看板视图 (4 个)**
- `GET /api/projects/{id}/kanban` - 看板数据
- `POST /api/tasks/{id}/move` - 移动任务（看板拖拽）
- `GET /api/projects/{id}/tasks/by-status` - 按状态分组
- `GET /api/projects/{id}/tasks/timeline` - 时间线视图

**子任务 (2 个)**
- `POST /api/tasks/{id}/subtasks` - 创建子任务
- `GET /api/tasks/{id}/subtasks` - 子任务列表

#### 数据库模型（整改后：2 张表）

**tasks 表（新增字段）**
```python
# 在 Phase 1 基础上新增：
- project_id: UUID (FK)
- assignee_type: ENUM('user', 'agent')  # 分配给人还是 Agent
- assignee_id: UUID  # user_id 或 agent_id
- status: ENUM('pending', 'in_progress', 'completed', 'approved', 'failed', 'rejected')
- parent_id: UUID (FK, nullable)  # 父任务
- position: INT  # 看板顺序
- due_date: DATETIME (nullable)
```

**task_comments 表**
```python
- id: UUID (PK)
- task_id: UUID (FK)
- user_id: UUID (FK)
- content: TEXT
- created_at
```

#### 前端页面（5 个）

- `/projects/:id/kanban` - 看板视图（核心）
- `/projects/:id/tasks` - 任务列表视图
- `/projects/:id/timeline` - 时间线视图
- `/tasks/:id` - 任务详情抽屉
- `/tasks/due-soon` - 即将到期任务

#### 前端组件

- KanbanBoard.vue - 看板主组件
- TaskCard.vue - 任务卡片（可拖拽）
- TaskDetailDrawer.vue - 任务详情抽屉
- TaskStatusBadge.vue - 状态徽章
- CommentThread.vue - 评论组件

#### 验收标准
- [ ] 看板拖拽排序正常
- [ ] 任务分配给人或 Agent 正常
- [ ] 任务状态流转正确（特别是 pending → in_progress → completed → approved）
- [ ] 评论功能正常

## Phase 5: 执行引擎集成（整改版）

### 定位
Phase 5 实现**任务触发 Agent 执行**，需要自建 WebSocket 服务。

### 整改点
- 明确 WebSocket 需要自建（不依赖 Hermes）
- 简化工作流概念为"任务链"（简单串联）

### 功能清单（整改后：~12 个 API）

**执行触发 (5 个)**
- `POST /api/tasks/{id}/execute` - 触发任务执行
- `GET /api/executions/{id}` - 执行记录详情
- `GET /api/tasks/{id}/executions` - 任务执行历史
- `POST /api/executions/{id}/cancel` - 取消执行
- `GET /api/executions/active` - 当前活跃执行

**任务链（简化版）(3 个)**
- `POST /api/task-chains` - 创建任务链
- `GET /api/task-chains/{id}` - 任务链详情
- `POST /api/task-chains/{id}/execute` - 执行任务链

**WebSocket 服务 (4 个)**
- `WS /ws/executions/{execution_id}` - 执行状态实时推送
- `WS /ws/tasks/{task_id}` - 任务状态实时推送
- `WS /ws/agents/{agent_id}` - Agent 状态实时推送
- `GET /api/executions/{id}/logs` - 执行日志（HTTP 备用）

#### 数据库模型

**executions 表**
```python
- id: UUID (PK)
- task_id: UUID (FK)
- agent_id: UUID (FK)
- status: ENUM('pending', 'running', 'completed', 'failed', 'cancelled')
- input_data: JSON
- output_data: JSON
- error_message: TEXT
- started_at: DATETIME
- completed_at: DATETIME
```

**task_chains 表（简化版）**
```python
- id: UUID (PK)
- name: VARCHAR(128)
- description: TEXT
- steps: JSON  # [{"task_id": "...", "agent_id": "..."}, ...]
- org_id: UUID (FK)
- created_by: UUID (FK)
- created_at
```

#### 前端页面（3 个）

- `/executions` - 执行记录页
- `/executions/:id` - 执行详情页
- `/task-chains` - 任务链管理页

#### 验收标准
- [ ] 任务能触发 Agent 执行
- [ ] 执行状态 WebSocket 实时推送
- [ ] 执行日志完整记录
- [ ] 任务链能串联执行
- [ ] 取消执行功能正常
