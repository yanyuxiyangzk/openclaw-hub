## Summary

## Issues Found

### 1. [HIGH] Agent Service - Stub Implementation

**File:** `backend/services/agent_service.py:139-184`

The `start_agent()` and `stop_agent()` methods only update the status field in the database but do not actually start/stop the Hermes agent process.

```python
def start_agent(self, agent: Agent) -> dict:
    """Start an agent - sets status to online"""
    agent.status = "online"  # Only updates DB, no actual start
```

Similarly, `get_agent_health()` and `get_agent_logs()` return hardcoded/simulated data:

```python
def get_agent_health(self, agent: Agent) -> dict:
    """Get agent health status - returns simulated health data"""
    return {
        "cpu_percent": 45.2 if agent.status == "busy" else 12.5,  # Simulated
```

**Impact:** Agent runtime management endpoints don't actually control Hermes agents.

**Recommendation:** Integrate with Hermes runtime or add TODO comments indicating stub implementation.

---

### 2. [MEDIUM] Project Soft-Delete Doesn't Cascade

**File:** `backend/services/project_service.py:72-74`

When a project is soft-deleted (`status='deleted'`), related `ProjectAgent` records are not cleaned up.

```python
def delete_project(self, project: Project) -> None:
    project.status = "deleted"  # ProjectAgents remain orphaned
    self.db.commit()
```

**Impact:** orphaned `project_agents` records for deleted projects.

**Recommendation:** Either soft-delete related records or explicitly delete `ProjectAgent` entries.

---

### 3. [MEDIUM] Project.settings Column Type

**File:** `backend/models/project.py:17`

```python
settings = Column(Text, nullable=True)  # Stored as TEXT (JSON string)
```

The `settings` field stores JSON as TEXT, but SQLAlchemy should use `JSON` type for proper serialization.

**Recommendation:** Use `Column(JSON, nullable=True)` instead of `Text`.

---

### 4. [LOW] Docker Compose Uses SQLite in Production

**File:** `docker/docker-compose.yml:11`

```yaml
DATABASE_URL=sqlite:///./openclawhub.db
```

The harness-context specifies PostgreSQL for production, but docker-compose uses SQLite.

**Recommendation:** Use PostgreSQL in docker-compose or add a development profile.

---

### 5. [LOW] Orphaned Import Statement

**File:** `backend/services/agent_service.py:1`

The file appears to have an import statement at line 1 that was concatenated incorrectly during file reading. This may be a display artifact but worth verifying.

---

### 6. [INFO] Frontend Store Response Handling

**Files:** `frontend/src/stores/agent.ts`, `frontend/src/stores/project.ts`

All API calls use `res.data.data` pattern:

```typescript
const res = await agentsApi.listAgents()
agents.value = res.data.data  # Assumes double-wrapped response
```

This assumes the API always returns `{ code: 0, data: { data: [...] } }` format.

**Recommendation:** Consider creating a unified API response handler.
